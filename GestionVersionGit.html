Gestion des Versions avec Git

Introduction au Versionnage

Git est un système de gestion de versions distribué qui permet de suivre l'évolution de ton projet au fil du temps. Grâce à Git, tu peux revenir à des états antérieurs de ton projet, collaborer efficacement avec d'autres développeurs, et garder un historique clair des modifications.

Cas de Figure : Imagine que tu travailles sur un projet de site web. Tu ajoutes des fonctionnalités régulièrement, mais un jour, une fonctionnalité introduit un bug. Avec Git, tu peux facilement revenir à une version précédente du site pour corriger le problème.

Commandes de Base pour Commencer avec Git

1. git init

Explication :
La commande git init est utilisée pour initialiser un nouveau repository Git dans le répertoire courant. Cela signifie que Git va commencer à suivre les changements des fichiers de ce répertoire. Lors de l'exécution de git init, un sous-dossier caché appelé .git est créé. Ce dossier contient tous les fichiers nécessaires pour suivre l'historique des versions, les configurations, et les métadonnées du projet.

Quand l'utiliser ?
Tu utilises git init lorsque tu veux commencer à utiliser Git pour un nouveau projet ou lorsque tu souhaites transformer un projet existant en un repository Git.

Exemple d'utilisation :

git init

Cette commande crée un repository Git vide dans le répertoire actuel.

2. git add .

Explication :
La commande git add . ajoute tous les fichiers et les modifications (fichiers modifiés, nouveaux fichiers, fichiers supprimés) de ton répertoire actuel à la zone de préparation (ou "staging area"). La "staging area" est comme une étape intermédiaire où tu peux rassembler tous les changements que tu veux inclure dans le prochain commit.

Détail :
Le . (point) signifie "ajouter tous les fichiers". Si tu souhaites ajouter un fichier spécifique, tu peux utiliser git add nom_du_fichier.

Pourquoi l'utiliser ?
Cela te permet de sélectionner les changements que tu veux enregistrer, tout en gardant les autres modifications en attente pour un futur commit.

Exemple d'utilisation :

git add .

Cette commande ajoute tous les fichiers et dossiers dans le répertoire courant à la zone de préparation.

3. git commit -m "message"

Explication :
La commande git commit -m "message" enregistre les changements ajoutés à la zone de préparation dans l'historique des commits. Chaque commit capture un instantané de l'état de tes fichiers à ce moment-là. L'option -m te permet de spécifier un message de commit directement dans la commande, entre guillemets. Ce message doit décrire brièvement les modifications apportées.

Pourquoi l'utiliser ?
Le message de commit est important car il explique pourquoi tu as fait ces changements, facilitant la compréhension de l'historique du projet. C'est une bonne pratique de rédiger des messages de commit clairs et informatifs.

Exemple d'utilisation :

git commit -m "Ajout de la fonctionnalité de connexion"

Cela enregistre les changements avec un message de commit décrivant ce qui a été fait.

Résumé des étapes :

  • git init : Initialise un nouveau repository Git.
  • git add . : Ajoute tous les fichiers modifiés, nouveaux ou supprimés à la zone de préparation.
  • git commit -m "message" : Enregistre les changements avec un message descriptif dans l'historique des versions.

Glossaire

  • Commit : Un enregistrement des modifications apportées à ton projet, avec un message descriptif.
  • SHA-1 : Un identifiant unique généré pour chaque commit. C’est un code de 40 caractères qui permet de retrouver un commit spécifique.
  • Mode Détaché (Detached HEAD) : Un état où tu es sur un commit spécifique sans être rattaché à une branche.
  • Branch : Une version parallèle de ton projet où tu peux développer de nouvelles fonctionnalités sans affecter la branche principale.
  • Staging Area : Une étape intermédiaire où tu places les fichiers que tu veux inclure dans le prochain commit.

Commandes pour Gérer les Commits

1. git log

Affiche l'historique des commits de ton repository, montrant le SHA-1, l'auteur, la date, et le message de chaque commit.

2. git show sha-1

Affiche les détails d'un commit spécifique, comme les changements apportés et le message du commit. Remplace sha-1 par l'identifiant unique du commit.

3. git checkout sha-1

Te permet de naviguer vers un commit spécifique pour examiner l'état de ton projet à ce moment-là. Tu seras en mode "détaché".

4. git checkout main

Revient à la branche principale, quittant le mode détaché pour reprendre le développement habituel.
