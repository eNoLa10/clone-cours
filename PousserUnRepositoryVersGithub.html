Créer un Repository sur GitHub et Pousser un Projet Local

Introduction

GitHub est une plateforme qui permet de stocker, gérer et partager ton code. Pour collaborer efficacement avec d'autres développeurs et garder un historique clair de l'évolution de ton projet, il est essentiel de savoir comment créer un repository et y pousser un projet local. Cette fiche explique comment faire cela étape par étape.

1. Créer un Repository sur GitHub

Pour créer un repository sur GitHub, suis ces étapes :

  1. Connecte-toi à ton compte GitHub.
  2. Clique sur le bouton New (ou le signe + en haut à droite) pour créer un nouveau repository.
  3. Donne un nom à ton repository, par exemple mon-projet.
  4. Choisis si tu veux que ton repository soit Public ou Privé.
  5. Ajoute une description (facultatif) et coche l'option Initialize this repository with a README si tu veux démarrer avec un fichier README.
  6. Clique sur Create repository pour finaliser la création.

2. Pousser un Projet Local sur GitHub

Pour pousser ton projet local vers GitHub, suis ces étapes :

  1. Ouvre Git Bash et navigue jusqu'à ton projet local :
    2. cd /chemin/vers/ton/projet
  2. Initialise un repository Git (si ce n'est pas déjà fait) :
    3. git init
  3. Ajoute tous les fichiers à la zone de préparation :
    4. git add .
  4. Crée un commit avec un message descriptif :
    5. git commit -m "Premier commit"
  5. Ajoute le lien du repository distant (copie le lien HTTPS de ton repository GitHub) :
    6. git remote add origin https://github.com/nom-utilisateur/mon-projet.git
  6. Pousse les fichiers vers GitHub :
    7. git push -u origin main

    Remarque : Si ta branche principale s'appelle master au lieu de main, remplace main par master dans la commande ci-dessus.

Glossaire

  • Repository : Un espace de stockage pour ton projet, contenant tous les fichiers et l'historique des modifications.
  • Commit : Un enregistrement des modifications apportées à ton projet, avec un message descriptif.
  • Remote : Un repository distant, comme celui hébergé sur GitHub.
  • Push : L'action d'envoyer des commits depuis ton repository local vers un repository distant.
  • Branch : Une version parallèle de ton projet, te permettant de développer de nouvelles fonctionnalités sans affecter la branche principale.
  • Main/Master : La branche principale d'un projet. Le nom main est souvent utilisé par défaut, mais certains projets utilisent encore master.
